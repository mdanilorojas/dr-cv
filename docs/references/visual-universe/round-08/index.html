01Six hero patterns observed across R1–R5

Every hero is a first contract with the visitor. Below are the six distinct opening patterns the research universe actually uses — plus a seventh that emerges from the overlap. Each card shows a sketched mini-hero, three real exemplars, and a one-line verdict for dr-cv against its "serio + cálido, agentic-designer, not maximalist" brief.

02Four section-rhythm archetypes annotated scroll maps

How do great landings pace light/dark bands, hero → proof → contact, or horizontal detours? Four rhythms recur across the universe. Each has a legitimate use — and a failure mode — for a site with dr-cv's content density.

Rhythm A · long-scroll product

Dark/light band alternation

The Huly / Linear / Vercel pattern. Dark hero → lighter band for trust (clients, testimonials) → dark for feature section → lighter for pricing/cta. Contrast creates chapter feel.

Hero · manifesto H1100vh
Logo marquee / live strip20vh
Proof band · numbers60vh
Feature cards · method90vh
Testimonials · cream70vh
CTA / footer50vh
prosSolved navigation for long content. Contrast creates rhythm. Accessible.
consRequires shipping two theme variants of every component — 2× CSS maintenance. Our recommended palette (Warm Huly, dark-only) resists the light bands.
Rhythm B · case-study scroll

Hero → case → case → case → contact

The Brittany / Rauno / Robin Noguier pattern. No features, no pricing. Short hero; cases are the entire middle; contact is one line at the bottom. Portfolio-native.

Hero · name + thesis80vh

Case 02 · /te-black100vh
Case 03 · EnRegla100vh

prosFits a portfolio-first brief perfectly. Every case gets its own viewport. Matches dr-cv's 4-case inventory exactly.
consAbout/timeline arrives late; first-time visitors may bounce before seeing the 15-year arc that explains the authority.
Rhythm C · tab SPA

Tabbed single-page app (current dr-cv v1)

One hero, then all sections collapsed behind tabs (Work / Method / About / Writing). The user picks the path. The Huly landing v1 already ships this.

Hero · H1 + thesis100vh
Tab nav · Work / Method / About / Writing10vh
Active tab content · swap in place~90vh
Footer · contact30vh
prosDense content without long scroll. Interactive feel. Already exists in landing-huly v1 so zero new cost.
consKills narrative arc — no ordering enforced, no story. Tab content often under-read. Bad for SEO / share-a-link scenarios where a specific case should be the destination.
Rhythm D · horizontal gallery

Horizontal-scroll detour

Vertical scroll transitions into a horizontal pinned section (Robin Noguier's case carousel, Active Theory scene swap). Spectacle. Breaks reader autopilot.

Hero100vh
→→→ horizontal · 4 cases slide by →→→300vh pinned
Method / about80vh
Contact40vh
prosMemorable. Cases feel like a museum hallway. Good for a small number of curated cases (exactly what we have: 4).
consR6 flags horizontal-pin as over-used / a11y-hostile / mobile-broken. Awwwards 2022–24 ran it to death. Skip.

05Narrative glue · one transition per proposal how sections connect

A scroll map is only half the story — the how of getting from one section to the next is what makes the page feel composed rather than assembled. One concrete transition recipe per proposal below.

Scroll-scrubbed

The artifact itself is the transition. verdict for dr-cv

Ship Proposal A · Restrained Huly.

Proposal A is the only proposal that absorbs Danilo's full content inventory — 4 cases, 2 active clients, a 15-year timeline, a method, two real testimonials, four numeric proofs — without compromising any of them. The restrained long-scroll gives each artifact its own viewport, and the "Warm Huly" palette chosen in R7 is dark-only by design, which is actually a feature here: it removes the light-band maintenance cost that Rhythm A normally imposes. A dark-throughout single-page scroll with warm accents carries the chapter change through typography and spacing instead of theme swaps.

Proposal B (editorial developer) is a close second and the most stylistically distinctive, but it reframes the brief: cases become essays, and the inline widgets (one per case) push implementation from ~1 week to 2–3 weeks. Worth keeping as a v2 evolution once traffic and conversion data justify the investment — the Emil Kowalski inline-demo pattern is the single most stealable move in the whole universe, but it belongs inside an already-shipped landing, not blocking the first one.

Proposal C (scroll-scrubbed artifact) is the one the Awwwards jury would notice — and the wrong bet for this phase. It requires 4–6 weeks of R3F plus glTF work (R6's HIGH-tier trap), it compresses 4 differentiated cases into timeline beats that flatten their distinctness, and it starves Phase 3 landing copy the moment it starts. Keep the "artifact as watermark" idea in the back pocket for a v3 "portfolio as object" pivot — it's a strong brand move that just isn't ready for ship.

  • Density fit. Danilo has 4 cases, 2 active clients, 15-year arc, method, 2 testimonials, 4 proof numbers — Proposal A gives each its own viewport without sacrificing any.
  • R6 verdict alignment. "Tier LOW is enough" — A ships on CSS + canvas 2D + Framer Motion; B requires inline interactive widgets; C requires WebGL.
  • R7 palette alignment. Warm Huly is dark-only by intent. A works natively with dark-throughout; Rhythm A's light-band requirement is obviated.
  • R5 winning techniques reused. A still steals the best of R5: bracket/slash section headings, numeric proof strip, one-sentence poster-adjacent H1 — without the gating and scroll-scrubbing that drag implementation out.
  • Narrative arc. Hero → marquee → work → method → about → contact is the Aristotelian three-act compressed into web grammar. Trust compounds; nothing is wasted.
ship plan Proposal A this phase · hold Emil-style inline widgets (from Proposal B) for v2 · hold scroll-scrubbed artifact (from Proposal C) for v3 once positioning is validated. Round 8 thus resolves to: warm dark long-scroll, manifesto H1 (Pattern B, paradox), Client marquee + live-data strip, inline-expanding cases, SVG-drawn method diagram, 15-year vertical timeline, inline testimonials, warm CTA glow at contact.
