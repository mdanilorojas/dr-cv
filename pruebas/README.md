## Subcarpetas

| Carpeta | Qué hay |
|---|---|
| `superpowers/specs/` | Specs formales (MD canónico + HTML companion para review) |
| `superpowers/plans/` | Planes de implementación por fase |
| `superpowers/visuals/` | HTMLs visuales de brainstorming, por `YYYY-MM-DD-<topic>/` |
| `superpowers/ideas/` | Exploración temprana |
| `landing-exploration/` | Variantes históricas de la landing (generators + tests descartados) |
| `prompts/` | Feedback verbatim del usuario guardado |
| `audits/` | Auditorías internas (GitHub, BAH capabilities, etc.) |

## Regla

Cuando no sé dónde va un archivo, cae aquí. Si después se consolida en producto, lo muevo. La duda no debe bloquear el flujo.

## Convenciones

- Visuales: un folder por sesión de brainstorming, fecha ISO al frente: `2026-05-12-case-card-redesign/`
- Specs+plans: pareja `foo.md` + `foo.html`, mismo basename
