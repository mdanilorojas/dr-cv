§ 1 Problem

The current v11 landing reads as a static catalog of "what I am": Agentic Designer, tagline, proof numbers, then cases. For the target reader — a technical hiring manager (Head of Eng / Head of Design) — that framing is wrong for two reasons:

  1. The job categories are mutating. Figma mastery and hand-coded UI — skills that used to define a role — are commoditized by agents. A senior reader knows this. A résumé built on those categories looks like a snapshot of the old world.
  2. The reader wants trajectory, not taxonomy. A hiring manager asks: "Does this person know where they're going, and is their time compounding?" Static titles and skill tags don't answer that; movement does.
What changes: reframe the landing from "what I am" to "where I'm investing my hours", with evidence.

§ 2 Target reader & success

Reader: Head of Engineering or Head of Design at a serious product company, scanning in 5 seconds and then reading for 30 if the scan earns it.

Success for this reader

  • 5-second scan: understands this person has made a deliberate shift away from commoditized craft toward orchestrated judgment.
  • 30-second read: sees concrete evidence of each move — clickable chips → real cases.
  • Walk-away phrase: "keeps the judgment, retires the hours" — or equivalent — as the summary they carry.

§ 3 Concept — Horizon Timeline

A three-stage horizontal timeline, placed immediately after the hero. Each stage is one column; the middle one is visually emphasized because that is where the reader lives.

Earned · 2010–2023
The craft I mastered, now commoditized

15 years shipping enterprise product. I keep the judgment — the hours that built it no longer compound.

Figma mastery Hand-coded UI Design reviews Component libraries Style-guide PDFs
Investing · 2024–Now
Where my hours go today

The frontline moved. I moved with it. Craft is now judgment + orchestration: I ship what used to take a team, and I ship the tools that make it possible.

Agent orchestration MCP servers Design-system-as-skill Recursive critic loops Taste as rubric
Next horizon · 12 mo
What I'm walking toward

Embedded in a serious product team, owning the seam between design judgment and agentic execution — where one person's leverage becomes a team's.

Team-scale agent tooling Audited agent workflows Eval-driven UI

The rule that keeps the future credible

A chip enters the Next horizon column only if it is (a) already backed by a public commit / skill / repo, or (b) explicitly marked as a declared bet with no evidence yet. No wish-list chips.

Tone rules

  • First person, direct verbs: I ship / I keep / I moved. No "I am X".
  • No role titles, no tag clouds, no taxonomy vocabulary.
  • Past is archived, not negated: "keep the judgment, retire the hours."
  • Future is concrete, never abstract aspiration.

Anchor prerequisites

The chips in §4 reference on-page anchors. Before implementation, the plan must audit the current v11 output and, for every anchor used, either (a) confirm it already exists, or (b) add an id attribute to the correct element (the case card, the Method tab panel). Anchor additions are the only permitted exception to "files untouched" — they are additive (id="…") and change no other markup. Anchors referenced: #case-enregla, #case-enregla-mcp, #case-te-skin, #case-te-black, #tab-method, #skills-audit. If a target doesn't exist and can't reasonably be added, the chip becomes a bet instead of evidence. Test #7 enforces this at build time.

  - id: horizon
    stage: { en: "...", es: "..." }
    heading: { en: "...", es: "..." }
    body: { en: "...", es: "..." }
    chips:
      - label: { en: "...", es: "..." }
        href: "https://github.com/mdanilorojas/dr-cv"
        kind: evidence         # enum: evidence | bet
      - label: { en: "...", es: "..." }
        kind: bet              # no href + kind: bet → dashed + badge

Accessibility

  • Non-clickable past chips: <span> with aria-label="Earned, now commoditized: {label}" so a screen reader contextualizes the strikethrough.
  • Clickable chips: native <a href>, keyboard-tabbable, visible focus using --v11-accent-ring.
  • "Bet" chips: aria-label="{label}. Declared bet, no evidence yet." + a visible small bet badge.
  • Decorative timeline line is aria-hidden.
  • Emphasis is multi-channel: accent color + larger dot + filled chip backgrounds — not color alone.

§ 8 Tests

Added to tests/landing/v11.test.ts.

  1. Section placement — renders after the hero and before the tab-nav, EN and ES.
  2. Emphasized column — investing column has data-emphasis="true" and the accent-dot class applied.
  3. Investing chips are links — all 5 chips in the investing column are <a> with non-empty href.
  4. Horizon column composition — the horizon column has exactly 2 chips kind=evidence + 1 chip kind=bet.
  5. Past chips have contextual aria-label — every chip in earned has an aria-label starting with "Earned" / "Ganado".
  6. HTML budget — rendered output stays below the 400_000-byte budget, both locales.
  7. Internal anchors resolve — every href="#..." points to an id that exists in the rendered HTML. Prevents silent dead links.

§ 10 Out of scope

  • No changes to the hero (thesis line + proof numbers stay).
  • No changes to other landings (dist/landing/, dist/landing-huly/).
  • No new "Direction" tab (explicitly rejected).
  • No deployment — ships as a local build only.
  • No new tokens, fonts, or external dependencies.

§ 11 Definition of done

  • npm run build:landing-v11 emits EN + ES under the 400k-byte budget
  • npm test passes (existing + 7 new tests)
  • npm run typecheck clean
  • Horizon section renders between hero and tab-nav in both locales
  • Investing column visually emphasized (accent dot + chip treatment + heading weight)
  • 5 past chips non-clickable with contextual aria-label
  • 5 investing chips are links to existing on-page anchors
  • 3 horizon chips: exactly 2 evidence (solid) + 1 bet (dashed + badge)
  • Mobile ≤ 920px collapses to vertical without overlap
  • Diff is surgical: hero, tabs, cases, testimonials, method, contact untouched
