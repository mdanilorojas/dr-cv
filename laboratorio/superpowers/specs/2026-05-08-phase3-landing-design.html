Goal

Ship a production-ready landing page at danilorojas.design (domain TBD) that:

  1. Loads the same /perfil/data/* single source of truth used by Phases 1 and 2 â€” zero hand-edited content.
  2. Matches the Huly.io aesthetic: dark-dominant, alternating light bands, JetBrains Mono display, Inter body, warm accent #FF8964, one animated visual per section.

Architecture

Tab-SPA pattern

One HTML file per language. All sections exist in the DOM at load; a tab bar toggles visibility via CSS [hidden]. URL hash reflects the active tab (/#work) for shareability. No router, no framework, no hydration.

Risks & mitigations

RiskMitigation
Tab-SPA hurts SEO vs multi-pageEach language gets its own <title> / <meta description>; hash URLs are client-only. Acceptable tradeoff.
Dark landing clashes with light CV/skills-sheetForked token set: tokens-landing.css separate from tokens-print.css. No cross-contamination.
Performance budget blown by fontsSubset to Latin-1 at build; woff2 only; font-display: swap. Fallback: drop JetBrains Mono.
Motion-sensitive usersprefers-reduced-motion: reduce disables non-essential motion; live-dot static.
Copy-email fails on iOS SafariFallback to window.prompt if navigator.clipboard unavailable.
Hash routing breaks browser backhistory.pushState + popstate listener syncs UI.
Lang switch loses active tabToggle href explicitly preserves location.hash.
