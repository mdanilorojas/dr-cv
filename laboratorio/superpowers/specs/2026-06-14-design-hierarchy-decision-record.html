Design hierarchy & precedence

Structure comes from MentorOS, looks come from Aura, and dark mode outranks both. Fixed now — while it is cheap — so future conflicts resolve without re-litigation.

Active 14 Jun 2026 Owner · Danilo Rojas Governs app · web · internal tools

The precedence ladder — highest wins

01

Dark mode

supreme

The default and the single highest-priority constraint. Overrides everything, including structure. Light mode extends the same semantic tokens — it never re-themes into a different brand; the accent only deepens to terracotta #D9663F to hold AA contrast on cream.

02

MentorOS — structure & hierarchy

source of truth

Information architecture: navigation, the screen set, grouping, the window.DATA contract, the flows. When MentorOS and the design system disagree about structure, MentorOS wins — and the decision is pushed into the design system.

03

Aura — visual language

in service

Tokens, color, type, spacing, radii, shadows, motion, components, and the signature artifact surfaces (embed · stream · plan · diff · skill-chip). Aura dresses the structure MentorOS defines; it does not redefine it.

How a structural conflict resolves

MentorOS proposes
A navigation / screen structure
Aura disagrees?
Its template suggests a different shape
Resolve toward
MentorOS wins — structure holds
Then
Push the decision into Aura; the DS updates to match
Hard exception: if the disagreement touches dark mode, rung 01 wins before this flow even runs. Dark is non-negotiable.

Worked example. Aura's career-os-app template ships a different nav (Workspace + Program). That is a structural disagreement, so MentorOS's four groups (Daily / Learn / Apply / Workspace) are canonical; the Aura template gets updated to match, not the reverse.

Two standing guardrails

No handwritten sans in interfaces

UI · never.  CV · maybe.  Blog background · candidate only.

Aura's stack stays Inter (UI/body) · JetBrains Mono (display/meta) · Source Serif 4 (essay only). This guardrail keeps a script face out of load-bearing UI text as the system grows.

The design system prioritizes UIs first

App and product surfaces get built and polished before marketing, landing, or print. When the polish budget is scarce, the app kit wins. App-first ordering on top of Aura's two-surface split.

Naming — so nobody gets confused later

  1. "The new design system" = Aura (codename dr-cv V11). Built from the repo's canonical design-system/tokens-web.css — same source of truth as the live landing.
  2. EnRegla is superseded. The embedded _ds/enregla-… system is dropped. On record: "dr-cv tokens only. No EnRegla." Kept for provenance only.
  3. MentorOS = Career Training OS = Career OS. Same app. "MentorOS" is the prototype; "Career OS" is the in-product brand string.
