Arquitectura

  • Nuevo generadores/templates/cv/clean.ts → renderCleanCv(data, lang).
  • cv.ts emite solo cv-clean-en y cv-clean-es (HTML + PDF A4 vía el pipeline puppeteer actual).
  • Paleta/tipografía inline en el template (self-contained, espejo del light de la landing). No depende de tokens-print.css (huly).
  • Data vía loadCvData(dataDir).
  • Archivo: sources viejos → laboratorio/cv-archive/; se quitan los emits y los outputs de dist/cvs/. Nada destructivo más allá de artefactos regenerables.

User flow — generar y usar el CV

Un cambio de contenido = editar perfil/data y reconstruir. El CV nunca se edita a mano en dist/.

Success criteria

  1. build:cvs emite exactamente cv-clean-en/es (HTML+PDF), sin warm/serious/bairesdev.
  2. Cada PDF = 1 página A4 exacta.
  3. Visual = dirección light aprobada.
  4. Sources viejos en laboratorio/cv-archive/; cv.ts ya no los referencia.
  5. npm test + typecheck verdes; tests de CV actualizados al nuevo template.

Fuera de alcance

  • Variantes por registro (sobrio vs AI-forward).
  • Anexo de casos / página 2.
  • Output dark.
