UR Training — Opciones de Arquitectura

80 preguntas en archivos Markdown locales → Life Update (React Native + Supabase)

A
Manual en el app
Bajo
Pros
  • Cero infraestructura nueva
  • Funciona hoy mismo
Cons
  • Tedioso actualizar a mano
  • Puede desincronizarse con el archivo real
  • No hay “cuál pregunta responder hoy”
B
Supabase table sincronizada
Recomendado Medio
Pros
  • App siempre sabe el progreso exacto
  • Puede sugerir “responde Q18 hoy”
  • Integración natural con misiones
  • Un solo script de sync al hacer review
Cons
  • Requiere crear tabla nueva + script de sync
  • Hay que correr el sync manualmente (o en CI)
C
GitHub API en tiempo real
Alto
Pros
  • Siempre lee el archivo real sin sync manual
Cons
  • Depende de conexión + GitHub token en el app
  • Latencia en cada carga
  • Complejidad extra en mobile
  • Acoplamiento frágil con GitHub API
▶

Opción B resuelve el problema sin sobreingeniería. Un script de sync que corre cada domingo en el review (o en CI si después se automatiza) alimenta la tabla Supabase que la app ya sabe leer.
