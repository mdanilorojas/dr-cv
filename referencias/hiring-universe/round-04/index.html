V11 · Round 4 · 2026-05-08 · dr-cv hiring universe

Case-study formats that convert

12 teardowns of real case studies from designers/engineers who got hired at Vercel, Linear, Radix, Klaviyo, Google, Stripe, and Figma-adjacent shops — plus 2 bootcamp-grade anti-patterns. What makes a hiring manager send the first message vs. close the tab.

The 3 case-study archetypes

After mapping 12 samples, every convincing case study falls into one of three shapes. The differences are not cosmetic — they signal different kinds of talent to the hiring manager.

Archetype A

The Artifact Drop

A single shipped thing, shown not explained. The page IS the case study.

  • Examples: emilkowal.ski experiments, rauno.me/craft, nerdy.dev CSS demos
  • Length: ≤ 300 words
  • Evidence: live interactive demo + code snippet
  • Signals: taste + craft floor
  • Risk: reads as "portfolio decoration" for PM/product roles
Archetype B · recommended

The Narrative Ship

Context → problem → approach → shipped result. Medium-length, metric-backed, team-credited.

  • Examples: brittanychiang.com projects, Josh Comeau course writeups, Maggie Appleton essays
  • Length: 600-1,500 words
  • Evidence: before/after + metric + quote + live link
  • Signals: shipped proof + collaboration + craft
  • Risk: takes effort to write well; medium-grade attempts feel bootcamp
Archetype C

The Manifesto-Study

A point of view dressed as a case study. Argues for a way of working, uses the project as proof.

  • Examples: Maggie Appleton tools-for-thought, ped.ro Radix notes, rauno.me/craft intro
  • Length: 1,500+ words
  • Evidence: prose + diagrams + selective screenshots
  • Signals: taste + agentic thinking + senior judgement
  • Risk: hiring manager bounces if they came for "did you ship"

Recommendation for dr-cv

Use Archetype B (Narrative Ship) as the default for the 4 Danilo case studies, with one Archetype C manifesto up top as the positioning piece ("how I work with agents"). AI Product Designer roles at tier-1 companies are hiring for judgement that shipped — not pure craft demos (Artifact Drop underweights collaboration) and not pure essays (Manifesto alone reads as a writer, not a shipper).

The manifesto becomes the handshake; the 4 narrative ships prove it's real.

Pattern reading the matrix

The 10 hired-at-tier-1 cases cluster on live demo + link to shipped + code snippet. Only one (Comeau) hits 8/8 because he's an educator — the rest optimise for "I made this real thing, touch it." The 2 anti-patterns hit before/after + process artifacts but miss the one that matters most: you cannot use their work. Lesson: the shipped link is the single most load-bearing evidence type.

Template for dr-cv case studies

Based on the findings above, here is the canonical shape for each of Danilo's four case studies. All four use Archetype B (Narrative Ship) with one concession to Archetype C: each opens with a 2-sentence positioning line ("how I work with agents on [x]") that accumulates into dr-cv's overall manifesto.

Shared scaffold (all 4 cases)

TITLE · [Role verb] + [artifact] for [client, anonymized if needed] DEK · 1 sentence. Names the outcome and the agentic leverage. META STRIP · Date range · Role · Team size · Stack · Link to shipped HERO · Live interactive OR 3-second looping video (not a static image) 1. CONTEXT (~120 words) Why this existed. Business stakes. 2. PROBLEM (~150 words) What was broken. Quote a stakeholder. 3. APPROACH (~250 words) How the agent + Danilo pair worked. Show 1 process artifact. 4. SHIPPED WORK (~300 words) Before / after. Live link. Code if public. 5. OUTCOME (~150 words) Metric + quote + what the client did next. 6. WHAT I'D DO DIFFERENTLY (~100 words) — non-negotiable. Signals senior. 7. CRAFT FLOOR STRIP a11y / perf / motion / keyboard — checklist style. Evidence target: Y on 6/8 types. Must hit: Live demo, Shipped link, Quote, Quant, Before/after, Process artifact. Length target: 900-1,300 words. Voice: first-person singular; "we" when credit is due; one admitted mistake; one sentence on the agent's role.

1 · EnRegla — compliance platform with paying pilots

Danilo's flagship product-judgement case. Leads with business outcome because "paying pilots" is the strongest signal.

Suggested verbatim title & dek

Title: "EnRegla — shipping a compliance platform with one designer and three agents" Dek: "Paying pilots in Ecuador's regulated industries, built by pairing me with Claude, Codex and a custom spec-lint agent."

Meta strip

2025—present Founder · Design + Product Team: 1 human + 3 agents Next · Supabase · TypeScript Live: enregla.com

Section checklist (EnRegla-specific)

1. CONTEXT The Ecuadorian compliance market + why SMBs can't afford BigCo tools. 2. PROBLEM Regulators change rules quarterly; spreadsheets fail. Stakeholder quote. 3. APPROACH How Danilo + agents split work: I do taste + judgement, agents do ship velocity. Show the spec-lint agent loop as a diagram. 4. SHIPPED Before/after: the paper-form baseline vs the live dashboard. Embed the actual dashboard (read-only demo account). 5. OUTCOME "Paying pilots" (no count, per NDA). Pilot quote if you can get one; else your own observation + numbers you CAN share (time-to-audit reduced from X days to Y). 6. DIFFERENTLY Overbuilt the admin panel in week 2; agents defaulted to feature-rich. Had to reassert the 80/20 cut. 7. CRAFT WCAG AA · <2s TTI on 3G · keyboard-first forms · offline pilot mode.

Evidence targets

Live demo (read-only) Shipped link Before/after (paper vs dashboard) Quote (pilot) Quant (time-to-audit) Process artifact (spec-lint loop diagram) Code snippet (optional) Video walkthrough (optional)

Length

1,200-1,400 words. Flagship case — permission to be longest.

DIFFERENTLY First draft of the Skill was too prose-heavy; agents still hallucinated. Had to re-cut as structured data + tiny runnable snippets. 7. Round 4 takeaways for V11

  • Archetype B (Narrative Ship) is the default for all 4 dr-cv case studies. Aim for 900-1,400 words each.
  • The single most load-bearing evidence type is "link to shipped". If you can't link it, embed a read-only demo.
  • Every case must include "What I'd Do Differently" (Comeau rule) — this is the senior-vs-mid discriminator.
  • Every case must include a craft-floor strip (Argyle rule): a11y / perf / motion / keyboard, checklist-style.
  • Credit collaborators by role, not name under NDA (Duarte rule, adapted).
  • Add epistemic-status chips (Appleton rule) on any section where the outcome is partial or pilot-stage.
  • Anti-patterns to refuse: speculative "Redesigning X" posts, Dribbble concept shots, solo "UX Designer" framing, by-the-book 9-section bootcamp templates.
