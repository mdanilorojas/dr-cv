Category A · 10 patternsDesign-portfolio anti-patterns

Compiled from public hiring-manager posts by Jehad Affoneh (VP Design, Splunk), Ridd (Planetscale), Jared Erondu (Lyft/Stripe), Julie Zhuo (ex-Facebook Design), bestfolios.com "worst of" archetypes, and 2024–2026 Design Twitter threads on portfolio first-impressions.

A-01

Process-theatre case studies

Every case opens with "Research → Synthesis → Ideation → Testing". Recruiter reads: this person confuses methodology with work. Dead giveaway the designer never owned the outcome.

Avoid Lead with the shipped artifact and the decision you made. Method surfaces only when it changed the decision.
A-02

Persona slides with invented photos

"Meet Sarah, 34, busy mom of two." Recruiter reads: bootcamp. Real product designers at IDEO/Figma quote research participants, not stock photos.

Avoid Use real quotes or skip the persona. Named pilot customer beats invented avatar.
A-03

Final-screen hero, no context

Tile grid of polished mockups with zero text. Recruiter reads: doesn't know why it looks like this. Craft without explanation reads junior at senior tiers.

Avoid Every hero tile needs one load-bearing sentence about the constraint that produced it.
A-04

"Redesigned [Airbnb / Spotify / Uber]" concept project

Unsolicited redesign of a Fortune-500 product. Recruiter reads: never worked inside real constraints. Airbnb's own designers have said this is an instant rejection signal.

Avoid If it shipped to zero users, it is a sketch. Label it that. Lead with work that has a URL an employer can visit.
A-05

Twelve case studies of equal weight

No hierarchy signals no taste. Recruiter reads: can't edit own work. Senior is defined by what you chose to cut.

Avoid 3–6 cases max, with one featured. Treat case count as a curatorial decision.
A-06

Every project credited to "I"

Contradicts the low-ego filter R1 found at Stripe/Mistral/Anthropic. Recruiter reads: can't name a teammate. Tier-1 hires name collaborators by the second paragraph.

Avoid Explicit role + team-size + what you specifically owned. "With [name] I owned X; they owned Y."
A-07

Dribbble-shot screenshots with no live link

Mockup in Figma space, no product URL. Recruiter reads: never shipped. At Linear/Vercel/Cursor this is disqualifying on its own.

Avoid Every case shows the shipped URL (even if internal/behind SSO, say so).
A-08

NDA as an excuse for a hollow case

"I can't show the work due to NDA." Recruiter reads: hasn't figured out how to talk about client work ethically. Senior designers anonymize and still ship insight.

Avoid Anonymize the client, describe the decision, quote the measurable outcome. BAH posture shows exactly how to do this (private not classified).
A-09

Manifesto-only home with zero artifacts

All vibe, no ship. Recruiter reads: thinker not doer. The Essayist archetype only works because there is named OSS below the fold.

Avoid Manifesto plus a named-artifact list (EnRegla, TE Skin, TE Black, Life Update, Aura). Not either-or.
A-10

Moodboards as portfolio items

Pinterest-grade aesthetic collages filed next to actual projects. Recruiter reads: dilutes the serious work around them.

Avoid Moodboards belong inside a case study as evidence, never as standalone tiles.

Category B · 10 patternsEngineering-portfolio anti-patterns

What staff engineers at Stripe, Vercel, Planetscale, Linear, and Cursor scan when a candidate claims "design engineer" in 2026. Pulled from leerob's hiring notes, Cassidy Williams on hiring, Mitchell Hashimoto's portfolio-review threads, and 2024–2026 Hacker News comment-threads on "what gets you hired".

B-01

GitHub profile with an empty contribution graph

Claims "ship in code" but the green squares are a desert. Recruiter reads: performance art, not practice.

Avoid Link to the repo that is the portfolio (mdanilorojas/dr-cv, EnRegla). Make the activity obvious.
B-02

Only toy-project repos, no shipped app

Twelve tutorials forks, no production code. Recruiter reads: learner not builder. Cursor/Vercel filter on deployed URLs.

Avoid Pin 4–6 repos that map 1:1 to named cases. Tutorials live in a clearly-labelled sandbox.
B-03

No README on pinned repos

Recruiter clicks, lands on a raw file tree. Instant close. A README is a writing sample.

Avoid Every pinned repo has a README with: what it is, a 90-second demo link, the decision it documents.
B-04

"Used React" as a claim in the bio

Naming the framework is 2014 energy. Recruiter reads: thinks tools are differentiators. At Vercel the framework is table-stakes.

Avoid Name the problem the code solved, not the framework. Stack belongs in a footnote chip, not the hero.
B-05

Portfolio site that fails Lighthouse

Your site is the one piece of code the recruiter will actually evaluate. A 40-performance score is a self-inflicted wound.

Avoid Ship the landing with LCP < 1.5s, CLS < 0.05, and self-host all fonts. Embed a Lighthouse badge if proud.
B-06

Package-dependency theatre

A simple landing that imports 40 npm packages. Staff engineer reads: can't taste simplicity. Anthropic's "do the simple thing that works" is the counter-filter.

Avoid Show the bundle size. Prefer vanilla HTML/CSS for the landing itself.
B-07

No commit history on the portfolio repo

One "initial commit, Next.js starter" then nothing. Reads: copied a template.

Avoid Commit the site the way you commit a product. Meaningful messages, incremental diffs.
B-08

view-source is broken / minified mush

In 2026 "view source" is a recruiting channel (R1/R2). If it is a single 900KB line of minified output, you just closed it.

Avoid Ship readable, commented, self-hosted HTML. Comments in source are a hireable signal.
B-09

Listing GPT-generated code as OSS

Repos with suspiciously-uniform commit cadence + no issues + no collaborators. Staff eng can smell it.

Avoid Be honest about what the agent wrote vs. what you designed. The "profile + agentic AI = combination" framing covers this cleanly.
B-10

Mismatch between claim and repo language

Bio says "Rust systems engineer"; every repo is a Next.js starter. Credibility wipeout.

Avoid Every skill claim must point at one public repo in that language. If it can't, downgrade the claim from "mastered" to "learning".

Category C · 12 patternsAI-specific anti-patterns, 2025–2026

The single noisiest category. Every portfolio in 2026 claims "AI". Most are riding the wave. Recruiters have calibrated fast — here is the short list of tells they now filter OUT.

C-01

"AI-powered" in the hero

Dead phrase in 2026. Recruiter reads: last year's pitch deck. Stripe's own listing says "Agentic Commerce", not "AI-Powered Commerce".

Avoid Use Agentic or name the agent (Claude, Gemini, v0). Never "AI-powered".
C-02

Buzzword stack: "AI / ML / GenAI / LLM / Prompt Engineer"

Five overlapping terms in a row. Reads as keyword stuffing for ATS. Anthropic's own copy uses none of these — it names behaviours.

Avoid Pick one precise frame ("agentic designer") and keep it. Let behavior and artifacts carry the rest.
C-03

"Prompt Engineer" as a title

The category collapsed in 2024. In 2026 it is a regression signal.

Avoid Say what you ship (skills, MCP servers, agent-consumable DS). Prompt-craft is an invisible substrate, not a title.
C-04

AI-generated hero illustration

The Midjourney dreamy-nebula hero is now a recruiter punchline. It signals no taste filter and no brand identity.

Avoid Ship a custom illustration, a photo, or a functional element — never a generic GenAI dreamscape.
C-05

Obvious GPT-generated case-study copy

"In today's fast-paced landscape…" / "I delved into…" / "leveraged synergies". Distinct LLM tells. Recruiter reads: doesn't write.

Avoid Writing voice should match how the candidate talks on a call. Ban "delve", "leverage", "robust", "seamless", "elevate".
C-06

AI claims without agent artifacts

Talks about AI for five paragraphs; nothing the recruiter can click and see an agent do. All sizzle, no steak.

Avoid At least one section shows an agent actually doing something (a /skill output, an MCP call log, a runnable demo).
C-07

Name-dropping models without context

"Used GPT-4, Claude 3.5 Sonnet, Gemini 1.5 Pro". Model names shift every 90 days. Reads as padding.

Avoid Name the capability (tool-use, MCP, skills, structured outputs), not the model SKU.
C-08

Replacing design taste with AI output

Portfolio that is 80% GenAI-styled screens + 20% real UI. Recruiter reads: can't tell good from generated. Cursor/Linear explicitly filter on "know what good looks like".

Avoid Human-designed decisions at top; AI output clearly framed as artifact, not as taste source.
C-09

Solo-genius framing with AI as a sidekick

"I 10x'd my output with AI". Recruiter reads: ego risk. Mirrors R1's disqualifier at Anthropic/Stripe/Mistral.

Avoid "Profile + agentic AI = combination." Leverage framing over lone-genius framing.
C-10

Fake agentic: chatbot wrapper called "agent"

A styled GPT chat with no tools, no memory, no multi-step plan. Recruiter clicks, sees a textarea, closes tab.

Avoid If you say agent, show tool-use, a plan, a version, an artifact (R3 vocabulary). Otherwise call it a chatbot.
C-11

2023-era AI screenshots (DALL-E 2, first GPT-4 UI)

Stale product surfaces. Recruiter reads: stopped paying attention 18 months ago.

Avoid Update product screenshots quarterly. The AI surface moves fast; the portfolio has to track.
C-12

Claiming "AI-native" with no MCP / skills / tool-use vocabulary

The 2026 fluency test (R3) is the vocabulary: artifact, thinking, skill, project, accept/reject, plan. Missing means behind.

Avoid Use the R3 vocabulary accurately. Name one skill, one artifact, one plan.

Category D · 10 patternsTone anti-patterns

Voice is a filter at the 20-second mark. Wrong register = close. Pulled from Design Twitter threads on portfolio voice, Jehad Affoneh's "portfolio tone" notes, and the Anthropic/Linear/Stripe copy-style comparison in R1.

D-01

"Just a simple designer who loves coffee"

Excessive humility at senior level reads as low confidence, not warmth. At Anthropic/Linear it undermines the evidence.

Avoid Warmth without self-deprecation. "Agentic designer shipping real products." is the calibration.
D-02

"World-class / rockstar / guru / ninja"

All four are fired terms in 2026. Self-applied superlatives read as insecure overclaim.

Avoid Let artifacts + testimonials carry the superlatives. Never self-apply.
D-03

Inconsistent voice: manifesto home, corporate case

Hero says "soulful craft"; case says "leveraged cross-functional alignment". Reads as two different writers.

Avoid One voice, whole document. If the case copy was written by a PM, rewrite it in your register.
D-04

Metrics without context ("10x growth")

"Grew engagement 37%". Of what? Over what? From what baseline? Reads as decorative, not measured.

Avoid Name the baseline, the delta, the window, and what it was attributable to. One good metric beats ten loose ones.
D-05

Consultant-speak ("stakeholders", "north star", "alignment")

BAH-origin vocabulary reads as process designer to a product company. Linear/Cursor/Vercel filter against it.

Avoid Swap "stakeholders" for "the PM and the two engineers". Swap "north star" for "the decision we bet on".
D-06

Bragging without specificity

"I delivered massive impact across multiple verticals". Recruiter reads: no specific proof.

Avoid Specific > superlative. "4 paying pilots on EnRegla" > "massive impact in B2B SaaS".
D-07

Apology for the current job

"Currently stuck at a consulting firm". Reads as ungrateful + unprofessional. BAH is a tier-1 US client; frame the leverage, not the cage.

Avoid Neutral description. "Design-engineering consultant at BAH via BairesDev" is the honest-proud register.
D-08

Over-branded personal tagline

"Crafting narratives at the intersection of humanity and tomorrow". Reads as agency bio, not hire.

Avoid Plain, declarative, operator voice. "Ships real products and the tools agents use to ship them."
D-09

Emoji clutter in headings

🚀🔥✨ on every section. Reads as LinkedIn thought-leadership, not senior IC.

Avoid Zero emoji in headings. Reserve them (if at all) for one specific playful signal.
D-10

False modesty about the AI itself

"Of course, the AI made mistakes, and I had to hand-hold it…" as a humble-brag. Reads as condescending + dated.

Avoid Respect the agent as a collaborator. Describe the capability, not its limitations.

Category E · 12 patternsVisual anti-patterns, 2026

What's tired, what's suspicious, and what reads as costume. Sourced from Rauno Freiberg, Emil Kowalski, Paco Coursey, Samuel Kraft threads, and Design Twitter's 2026 "please stop doing this" roundup.

E-01

Gradient-mesh hero blob

2022 fatigue. Every AI-startup landing had one; in 2026 it reads as template.

Avoid Either kill the mesh or make it functional (e.g. dataviz). No decorative blobs.
E-02

Huly-clone warm-radial-glow pill CTA

Every AI-infra landing in 2025 shipped this pill. In 2026, recruiters who browse portfolios daily recognize it in 0.5s as copy.

Avoid If the pill stays, make the glow cooler, the easing different, or replace with a flat button. Subvert the mimicry.
E-03

One-off decorative WebGL sphere

A spinning torus that does nothing. Costs LCP, earns nothing.

Avoid WebGL only if it demonstrates a skill (dataviz, 3D DS preview). Otherwise CSS or canvas 2D.
E-04

Chromatic aberration on everything

RGB-split on every image. Noise mistaken for taste.

Avoid Zero chromatic aberration. If retained, use it on one load-bearing element.
E-05

Scroll-jacking

Overriding native scroll. Recruiter reaches the bottom in 2x the time; half bail. Measurable drop-off.

Avoid Native scroll, scroll-behavior: smooth is the ceiling. No pinned-cinematic sequences.
E-06

Loading screen > 2 seconds

In 2026, > 1.5s LCP is failure. A "loading portfolio" animation is self-reporting the problem.

Avoid Zero loading screens. Show content immediately, lazy-load below the fold.
E-07

Forced horizontal scroll sections

Gallery requires sideways scroll via keyboard. Hostile on trackpads, impossible on mobile.

Avoid Vertical scroll is the standard. Horizontal gallery only as optional flourish, never the primary path.
E-08

Cursor-following blobs / custom cursors

Breaks accessibility, breaks native selection. Reads as 2019 Awwwards submission, not 2026 hire.

Avoid Native cursor. Subtle hover states do the signaling.
E-09

Auto-playing video hero with sound

Catastrophic on a recruiter open-desk scan. Instant close.

Avoid Muted, auto-pause on tab-blur, prefers-reduced-motion aware, never sound.
E-10

Excessive easter eggs / konami-code secrets

Clever to yourself, invisible to 99% of viewers. Content effort displaced into novelty.

Avoid One subtle easter egg is charm. Three or more is look at me.
E-11

Mobile that forgot to exist

Desktop-perfect, mobile broken. 60% of recruiter opens are phone-first in 2026 (LinkedIn mobile → browser).

Avoid Test 375px width weekly. Mobile is the primary, not the fallback.
E-12

Dark mode only, no light-mode affordance

Some recruiters screenshot into light-themed Notion. Dark-only can wash out on screenshot + paste.

Avoid Either provide a toggle, or ensure the dark design reads legibly when screenshotted and recoloured.

Category F · 10 patternsStructural anti-patterns

Information-architecture tells. Recruiters do not read your site — they scan it. Structure is a hireable signal.

F-01

"About me" before showing work

The biography-first portfolio reads as introvert, not operator. Every portfolio in R2 opens with a shipped artifact or manifesto, not a bio.

Avoid Hero → Work → Method → About. Bio lives at the bottom.
F-02

Twelve case studies

More than 6 is not curated. Tier-1 hires showed 3–5 (R2).

Avoid 3–6 max. Everything else to an "archive" page.
F-03

Two case studies

Too few is no range. Senior designers can show 4 different problem shapes.

Avoid Under 3 cases reads junior. Keep to 3–6.
F-04

Case studies without outcome

"Here's the design" — no mention of ship, pilots, metrics, or retrospective. Reads as a deck, not work.

Avoid Every case ends with an Outcome block: shipped / pilots / metric / what we learned.
F-05

Dead links

Case link 404s. Single worst signal — broken basics.

Avoid Link-check in CI. A dead project link is a rejection in 2s.
F-06

Missing target role

Lesson 5 from R2: the target role goes on the site. Missing it means recruiter has to guess. Guesses default to rejection.

Avoid State the target role explicitly in the hero or chrome: "AI Product Designer / Agentic UX Lead".
F-07

Resume link that 404s

Portfolio says "View full resume" → 404. Evidence of zero maintenance.

Avoid The CV PDF lives in-repo, versioned. Link to the latest always resolves.
F-08

No contact affordance above the fold

Recruiter loved the work, can't find an email in 5 seconds, tabs closed.

Avoid Email in the nav or hero. A copy-button beside it.
F-09

Contact form with no fallback

Form without visible email. Recruiters trust email, not your form.

Avoid Email first, form optional. Always show the email address.
F-10

No availability state

R2 Lesson 4: explicit "open / closed / selective" magnetizes. Middle is worst.

Avoid Persistent availability pill: Open to AI Product Designer / Agentic UX Lead roles.

Category G · 10 tellsThe bootcamp-portfolio tell

10 cumulative giveaways. One of these is forgivable; 3+ stacked are disqualifying at senior tier. Sourced from General Assembly, Springboard, Designlab critiques and ex-bootcamp-grad confessionals, cross-referenced with Jared Erondu / Julie Zhuo hiring-screen notes.

G-01

"Hi, I'm [name] and I help brands tell their story"

Cohort-issued opener. Reads as template fill-in.

Avoid Concrete claim-of-work as the opener.
G-02

"UX/UI Designer" as the title

The slash is a bootcamp artifact. Senior designers say Product Designer, Design Engineer, Interaction Designer, etc.

Avoid Pick one specific role category. "Agentic Designer" is fine; "UX/UI Designer" is not.
G-03

Double-diamond diagram

The most-recycled process visual in the world. Its presence is the signal.

Avoid If process matters, custom-draw the actual flow. No double-diamond, no DDS.
G-04

"HMW" statements as case anchors

"How Might We" framing is IDEO-classroom syntax. Outside IDEO it reads as classroom.

Avoid State the actual product decision. "We chose X over Y because Z."
G-05

A "Tools I use" section with 20 logos

Figma/Sketch/Adobe XD/Framer/InVision/Principle… Tools are not differentiators.

Avoid Drop the tool-logo wall. Stack belongs in one footer line if at all.
G-06

All projects from the same 3-month window

Cohort cadence. Recruiter sees the dates clustered, reads bootcamp cohort, not career.

Avoid Spread the portfolio across years (Danilo has 15 — use them). Real career timeline beats cohort snapshot.
G-07

Fake client logos

"Projects for" Netflix/Spotify/Uber without contract. Legally dubious, reputationally fatal.

Avoid Only real clients. "Concept / spec work" if unsolicited.
G-08

Stock photography on hero

Unsplash gradient, diverse-team-high-fiving. Generic-AI aesthetic before AI even rendered it.

Avoid Custom illustration, real product screenshots, or text-only hero.
G-09

No shipped URL on any case

All Figma prototypes, no production. Pattern matches bootcamp capstone.

Avoid At least half of cases link to a live (or internally-linked) product surface.
G-10

Heavy use of "I learned" in every case

Senior designers ship; they don't narrate what they learned. "I learned" as a recurring refrain reads junior.

Avoid One short "what this changed about how I work" line per case, not "I learned" paragraphs.

Category H · 10 patternsMeta anti-patterns

Hygiene that has nothing to do with design but kills the candidacy anyway. 90% of rejections at the silent-filter stage are in this category.

H-01

Broken on mobile

Recruiter opens from LinkedIn on phone. Broken = closed.

Avoid 375px-first layout. Test weekly on a real phone.
H-02

LCP > 3s

In 2026, 1.5s is table-stakes. >3s = can't ship.

Avoid Self-host fonts, inline critical CSS, lazy-load below fold. Measure weekly.
H-03

External-CDN font dependency that stalls

Google Fonts CDN blip => FOIT and CLS. Self-inflicted.

Avoid Self-host Inter + JetBrains Mono. Use font-display: swap + preload.
H-04

Stale — last case 18+ months old

Recruiter reads: hasn't shipped lately. Fatal in AI where 6 months is a generation.

Avoid "Currently" pill surfaces current work. Update monthly. Date-stamp cases.
H-05

Missing contact methods

No email, no LinkedIn, no X. Uncontactable = unhirable.

Avoid Email + LinkedIn + GitHub, all visible in contact section + footer.
H-06

Broken image / missing-asset squares

One broken image signals doesn't QA own site.

Avoid Image audit in CI. onerror fallback on every <img>.
H-07

No favicon / default framework favicon

The Next.js "N" favicon tells the recruiter didn't finish polishing.

Avoid Custom favicon. Brand the tab.
H-08

Missing OG image / Twitter card

Link shared in DM / Slack shows no preview. Every share is a recruiting surface.

Avoid OG image + Twitter card with title + subline. Test with metatags.io.
H-09

No HTTPS

http:// subdomain in 2026. Browser warns. Recruiter never gets past the warning.

Avoid HTTPS only, HSTS preload. Vercel/Netlify handle this for free.
H-10

No accessibility basics

No alt text, no focus states, fails axe. At Anthropic/Figma/Airbnb a11y is a hiring filter.

Avoid Alt on every image, visible focus rings, skip-to-content link, axe/Lighthouse a11y ≥ 95.

Final-final · V11 build-tapeTop 10 anti-patterns to absolutely avoid in V11

Distilled from the full matrix. These are the 10 highest-signal, highest-risk anti-patterns for a 2026 recruiter at Anthropic / Stripe / Linear / Figma / Vercel opening dr-cv for the first time. Post this as a checklist above the V11 editor.

01

Do not ship the Huly-clone warm-radial-glow pill CTA. (E-02)

In 2026 this silhouette is recognized as template in 0.5s. Flatten the pill, cool-shift the glow, or replace entirely. This is dr-cv's single highest visual risk today.

02

Do not claim "agentic" without a running agent artifact on the page. (C-06, C-10)

Saying it without showing it is the defining 2026 disqualifier. Embed a /te-skin output preview, an MCP call log, or a live component render above the fold.

03

Do not omit the target role. (F-06, R2 L5)

"Open to AI Product Designer / Agentic UX Lead" must appear in chrome, not just as a green dot. Recruiters filter on role category, not on vibe.

04

Do not load fonts from a third-party CDN. (H-03)

Google Fonts stall ⇒ FOIT + CLS ⇒ LCP fail. Self-host Inter + JetBrains Mono as woff2. The landing is the one piece of engineering work the recruiter will evaluate.

05

Do not write case-study copy that smells like GPT. (C-05, D-05)

Ban "delve", "leverage", "robust", "seamless", "elevate", "stakeholders", "north star", "alignment". Run every case through the ban-list before shipping.

06

Do not ship metrics without baseline + window + attribution. (D-04)

"4 paying pilots at EnRegla" > "massive impact". Every number must answer: of what, over what, from what baseline. One good metric per case beats a dashboard.

07

Do not frame yourself as the lone genius leveraging AI. (C-09, R1 Must-05)

Anthropic / Stripe / Mistral filter against the sole-genius narrative. Hold the "profile + agentic AI = combination" line. Testimonials must praise method + leverage, never sole talent.

08

Do not let mobile break, and do not let LCP exceed 1.5s. (H-01, H-02, E-11)

60% of first opens are phone-first via LinkedIn. Broken mobile or slow LCP = instant close, no matter how good the desktop is. Test 375px weekly.

09

Do not ship a portfolio that is stale. (H-04)

Date-stamp every case ("Last shipped: 2026-05-07"). Surface a "Currently" pill ("Currently: EnRegla pilots + TE Developer Portal"). In AI, silence > 3 months reads as exit.

10

Do not hide the writing. (R1 Must-02, A-09)

Anthropic verbatim: "put that at the top of your resume." One long-form essay (agentic-designer manifesto, EnRegla postmortem, or BAH DS methodology anonymized) surfaced at case-tile weight. Writing is the single most under-used differentiator for dr-cv competing with US ICs.
