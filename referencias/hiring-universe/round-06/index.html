Part 1Six categories of verifiable proof

Across R1 (what employers say) and R2 (what their hires actually publish), six proof types do all the work. Anything that cannot be placed in one of these buckets is decoration, not proof.

A
Shipped product
Something live that accepts public traffic today. The single most load-bearing category — R4's evidence audit showed "link to shipped" as the one signal every hired-at-tier-1 case study carried.
Landing URL on custom domain GitHub repo with stars & README npm / App Store / Chrome Web Store listing 60-sec demo video, captioned
Used by: 14 / 14 hires
B
Agentic-specific
Repos, skills, or MCP servers that use or build with AI agents. Distinct from "AI product" — this is the category Linear, Stripe, and Figma now hire into by name ("Agentic Commerce", "AI Models", "agents and teams").
MCP server on awesome-list Published Claude-Code skills repo .riv / Rive files showing agent loops Screen-capture of a live subagent run
Used by: 3 / 14 hires · scarce, so differentiating
C
Writing
Anthropic names it explicitly: "if you've written a thoughtful blog post, put that at the top of your résumé." Essays, threads, talks. This is the proof that survives when the artifact is NDA-locked.
Essay, 800+ words, own domain Thread, 100+ reactions, quotable Talk, conference or community, YT link Podcast appearance with transcript
Used by: 9 / 14 hires · Essayist cluster
D
Community
Public traces of working-in-public. GitHub activity graph, Discord moderation, build-in-public log. Less weighted than A–C but cheap to accumulate and compounds over weeks.
GitHub contribution graph, 90-day density PR reviews / maintainer badge on OSS Build-in-public log (changelog page) Office hours / mentoring trail
Used by: 11 / 14 hires (ambient)
E
Credentialed
Named employers or clients, published case studies by the client, awards, conference slots. High signal per instance — but slow to acquire and often NDA-hostile. Worth exactly what it says on the tin.
Tier-1 client namedropped (Stripe, Booz Allen) Client-published case study Awwwards / CSS Design Awards / FWA Verified testimonial on LinkedIn
Used by: 8 / 14 hires
F
Numeric
One load-bearing number per artifact. R2 lesson 03: resist the dashboard. Stars, installs, commits, uptime — but only when attached to a named shipped thing. Percentages without a named artifact read as consulting-speak.
Stars / forks / npm weekly downloads Commit graph / days-active Lighthouse / WCAG / bundle-size Paying users / MRR (when namable)
Used by: 3 / 14 hires · rare, always specific
Filter rule. A proof is only in-scope if it is (1) public, or (2) verifiable by a recruiter in under five minutes. Private Figma links, unlabelled screenshots, and Medium drafts fail both tests. Everything in R6 is scored against the five-minute rule.
