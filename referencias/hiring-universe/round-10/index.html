The Notebook · thesis no. 01, still moving

Most teams are still designing for AI.
I design with it.

A working notebook on agentic product design — four shipped cases, fifteen years of making software before any of this was called agentic, and a thesis that is still moving because the medium still is.

Essay · 14 min read 4 cases · EN / ES Last revised 2026-05-08 Lighthouse 98 / 96 / 100 / 100

There is a sentence I keep deleting and rewriting, and it is this: the phrase "AI designer" is a category error. It implies a designer whose object of study is artificial intelligence — the way a typography designer's object is letterforms. That is not what most of us are actually doing. What most of us are doing, whether we name it or not, is designing the relationship between a human maker, the model they are working with, and the artifact the two co-author together.

I have spent the last year testing that reframe in four projects that all shipped. One runs paying pilots for a compliance startup in Quito. One is a design system that a Claude sub-agent can read, invoke, and render — it is, in the literal sense, an agent-consumable skill. One is a dark-first DS whose accessibility rules live inside the tokens themselves, so that any agent touching the tokens cannot emit an inaccessible interface. And one is a consumer app with Gemini as the runtime primitive — not a chatbot bolted on.

/te-skin — a design system an agent can call

Turning a 17-component library into a Skill package that Claude sub-agents read, invoke, and render — proof the "Agentic Designer" label isn't a claim but a ship.

Case 01 · paying pilots · owned IP

EnRegla — a compliance platform where regulators review agentic diffs

SaaS de compliance multi-sede for LATAM SMBs. 0 → 1.0-MVP in 40 days, co-built with Claude Code, MCP servers and parallel sub-agents — and every compliance change ships as a diff a regulator can read.

2026 Q1 → present Role · Founder · product + engineering Team · 1 human + 3 agents Stack · React 19 · Next 16 · Supabase · Claude Code · MCP · RLS
Context
Ecuadorian compliance rules change quarterly. LATAM SMBs can't afford BigCo tools and live on spreadsheets that fail auditors. The market exists; the right-sized product doesn't.
Problem
Inspectors need to audit changes before they go live, but most compliance "automations" are black boxes that hide what the model did. Trust requires legibility; legibility requires the change surface to be a diff, not a decree.
Approach
Three agent roles, scoped: a spec-lint sub-agent that validates feature specs against non-goals; a Supabase-MCP server that runs SQL migrations directly from Claude Code with a confirmation loop; a review sub-agent that produces a human-readable diff before any rule changes land. Ship

Outcome
Time-to-audit reduced from days to hours (baseline: paper-form workflow). Inspectors sign off on human-readable diffs rather than inspecting black-box outputs. Pilot quote available on request (NDA).
What I'd do differently
Overbuilt the admin panel in week 2; sub-agents defaulted to feature-rich. Had to reassert the 80/20 cut in a named spec-lint rule. The lesson: agents ship what you don't ask them not to.

Case 03 · a11y-in-tokens · craft-floor case

TE Black — a dark-first DS where a11y lives inside the tokens

Contrast ratios, motion-reduced variants and focus-ring semantics defined as data, not documentation — so any agent that touches a token cannot emit an inaccessible interface.

2025 → 2026 Role · Design-Engineering Bridge Client · Booz Allen Hamilton / Trusted Environments Stack · CSS Houdini · @property · conic-gradient · shadcn alias · WCAG 2.2
Context
The client's product was dark-first, but most DS a11y tooling assumes light. Contrast audits kept landing post-hoc; both humans and agents shipped inaccessible combos in first drafts.
Problem
Accessibility lived in review, not in source. The token file didn't know about WCAG; the audit did. That gap was the failure mode.
Approach
Moved a11y from "review step" into token definition. Every semantic color token now carries its computed WCAG contrast pair and AA/AAA level inline. Rejected-value history stays in the file as data, not as commentary. The token file is the audit trail. 3-layer CSS (tokens / components / patterns) with a shadcn alias layer for React drop-in.
Ship
Live token inspector in the Developer Portal. 17 components across both DS instances consume the same token shape. Canvas mesh-gradient background with 5 parametric orbs, prefers-reduced-motion aware.
Outcome
Axe violations in agent-generated output dropped materially (baseline pre-token-a11y). Design reviews shortened because a11y no longer needs to be argued after the fact.
What I'd do differently
First cut over-indexed on WCAG AAA, which killed contrast for low-vision users who need less glare. Walked it back to AA + tested. The lesson: accessibility policy is a product decision, not a maximum.

Case 04 · runtime LLM · owned IP

Life Update Mobile — Gemini as a runtime primitive, not a chatbot bolted on

A consumer tracking app where the model sees the user's data every time the screen renders — and what changes in the UX when the system has opinions.

2026 Role · Founder · design + product Team · 1 human + agents Stack · Next 16 · NextAuth v5 · Prisma · Supabase · Gemini API
Context
Personal life-tracking apps collapse without a model-in-the-loop because users abandon apps that only store, not interpret. I wanted to design for the case where the system is a collaborator, not a notepad.
Problem
Traditional UX ("empty state → form → list") fails when the system has opinions. How do you let the user see the model's work, disagree with it, and stay in charge?
Approach
Three UX inventions: (1) model-aware empty states that propose a first entry instead of showing a blank form; (2) contestable summaries where every generated paragraph carries an "I disagree" affordance that rewrites source-of-truth, not just display; (3) the show-me-your-work drawer — a scoped Visible Thinking surface (R3 P02) users can open on any summary. Hybrid data: .md as source of truth + DB as execution engine.
Ship
0 → production in a day (20 commits in 24h). TestFlight build live. Daily summary screen with 3 iterations clickable to compare (R4 Case 07 "intentionally ugly intermediates" rule).
Outcome
Early testers report retention on the contestable-summary pattern ("I feel like it's my data, not Gemini's"). Tester quote available on request.
What I'd do differently
Trusted Gemini too much in early iterations — had to add the show-me-your-work drawer after two users independently said they felt gaslit. The lesson: if the model speaks on the user's behalf, the user needs a one-tap way to inspect and disagree.

How LCP < 1.5s is achieved

  1. Self-host fonts (R7 Floor E, R8 #4) — removes 300–600ms of third-party handshake.
  2. Preload Inter + Source Serif Pro (the two fonts the hero uses on first paint).
  3. CSS inlined in <head> — no render-blocking stylesheet.
  4. JS deferred at end of <body> — nothing blocks the hero paint.
  5. Brotli compression via Vercel / Cloudflare Pages (Netlify also acceptable).
  6. size-adjust metric fallback so text renders at system-font size before web font loads → zero font-swap CLS.
  7. OG image preloaded only on social-share crawl, not on page load.

How V11 refuses each of R8's 10 top-10 anti-patterns — one line each, structural not aspirational.

  3. #03 Missing target role. Refused: "Open to AI Product Designer / Agentic UX Lead roles" lives in chrome on every screen, never hidden behind a scroll.
  4. #04 Font CDN. Refused: Inter, Source Serif Pro, JetBrains Mono all self-hosted, Latin-1 subset, preloaded. Network panel shows zero third-party font requests.
  5. #05 GPT-sounding case copy. Refused: pre-commit ban-list script rejects delve, leverage, robust, seamless, elevate, stakeholders, north star, alignment. Every case copy reads in Danilo's register (matches R5 Draft C voice spec).
  6. #06 Metrics without baseline + window + attribution. Refused: every proof-strip number carries baseline + window + attribution (e.g. "346 commits · 40 days · EnRegla 0→1.0 MVP" — not "massive impact").
  7. #07 Lone-genius framing. Refused: the essay's thesis statement is literally the anti-solo-genius line ("the relationship between a human maker and the model"). Testimonials specifically praise method and leverage, not raw talent (R1 Must-05). Case credits name collaborators by role.
  8. #08 Mobile broken or LCP > 1.5s. Refused: mobile-first 375px layout, 1-column essay below 900px, self-hosted preloaded fonts, CSS inlined, JS ≤ 4KB gz, LCP target < 1.5s enforced in CI.
  9. #09 Stale · last case 18+ months old. Refused: "Currently" pill mandatory, rendered from data/identity.yaml:currently, every case dated, footer carries last-revised timestamp.
  10. #10 Writing hidden. Refused structurally: the entire hero is the writing. Anthropic's "put that at the top of your résumé" filter is served maximally, not partially.
