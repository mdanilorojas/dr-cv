The 2026 Agentic UX design patterns

Ten patterns observed in two or more shipped products. These are the building blocks a hiring manager in 2026 expects an AI Product Designer to know by name.

P01

Artifact Panel (split-pane chat + output)

Long-form AI output lives in a persistent right panel, never inline bubbles. Chat becomes the controller for the artifact.

Exemplars: Claude Artifacts, v0, OpenAI Canvas, Figma Make, Lovable
P02

Visible Thinking, collapsible by default

Chain-of-thought streams in real time but stays collapsed unless the user expands it. "Thinking" is the dominant label.

Exemplars: Claude, Perplexity Pro, Cursor, ChatGPT o-models
P03

Tool Call as a Card

Every tool invocation renders as a compact card with name, args, status, and collapsible result. Users can audit what the agent did.

Exemplars: Claude Code, Cursor Agent, Bolt, Replit Agent
P04

Plan before Build (editable plan)

Agent externalises a plan or spec as an editable artefact; user can edit before execution, steering the run.

Exemplars: Copilot Workspace, Claude Code, v0, Bolt
P05

Diff-Review as the Unit of Trust

Every AI mutation lands as a reviewable diff or versioned change. Nothing applies silently.

Exemplars: Cursor, Copilot Workspace, Claude Code, v0
P06

Checkpoint & Rollback

Long autonomous runs write named checkpoints; users can roll back the entire world (files, DB, deps) to any prior state.

Exemplars: Bolt, Replit Agent, Lovable
P07

Sources-first Answering

Citations or retrieved sources precede the answer, with inline numbered references inside the prose.

Exemplars: Perplexity, Notion AI Workspace Q&A, Claude with web search
P08

Selection-as-Prompt (localised AI action)

Select a region (text, component, block) to invoke AI scoped to that selection. Replaces "write a long prompt".

Exemplars: OpenAI Canvas, Figma Make, Notion AI, Cursor
P09

Named Capabilities (Skills & @-mentions)

Tools are exposed to the user as named, composable skills — invoked explicitly via @-mention, slash command, or skill picker.

Exemplars: Claude Skills, Raycast AI Extensions, Cursor MCP, Perplexity (Search / Pages)
P10

Memory Split: Instructions vs Knowledge

Persistent state is decomposed into (a) instruction / style / role text, and (b) pinned documents or files. Two different shelves.

Exemplars: Claude Projects, ChatGPT Projects + Memory, Cursor Rules + indexed repo

What dr-cv's case studies should emphasize

Mapping Danilo's four case studies onto the patterns and vocabulary above, so the landing demonstrates fluency. Each case gets: the right term, the right pattern, and a one-line framing.

EnRegla — compliance platform

Framing · Agent-assisted compliance workflows

Vocabulary to use

  • Call the AI features skills, not "AI features".
  • Call the outputs artifacts (compliance reports, driver packets).
  • Call long runs agentic workflows, not "automations".

Patterns to emphasize

  • P03 Tool Call as a Card — audit trail for regulators.
  • P04 Plan before Build — editable plan before batch compliance runs.
  • P05 Diff-Review as the Unit of Trust — inspectors sign off on diffs.
Headline pattern: “Plan → Review → Apply” — compliance changes ship as agentic diffs, not black-box automations.

TE Skin — agent-consumable Design System

Framing · Design System as a Skill

Vocabulary to use

  • Call it a Skill (agent-consumable) — not "component library".
  • Talk about tokens as tool inputs, not "design tokens".
  • Call the output agent-rendered artifacts.

Patterns to emphasize

  • P09 Named Capabilities — TE Skin IS a named skill agents invoke.
  • P08 Selection-as-Prompt — designers scope changes inside Figma.
  • P10 Memory Split — DS docs are Knowledge, tokens are Instructions.
Headline pattern: “A Design System that an agent can call” — the DS is not a library humans read, it's a skill humans and agents both invoke.

TE Black — dark-first DS with a11y in tokens

Framing · Policy-as-tokens

Vocabulary to use

  • Call contrast/a11y rules policies encoded in tokens.
  • Call the audit a token-level a11y skill.
  • Avoid "component audit" — use automated a11y run.

Patterns to emphasize

  • P03 Tool Call as a Card — a11y audit outputs a readable report.
  • P05 Diff-Review — contrast fixes ship as reviewable diffs.
  • P10 Memory Split — accessibility rules live in Instructions layer.
Headline pattern: “Accessibility as a skill the agent runs” — dark-mode + a11y is not a checklist, it's a capability invoked on every token change.

Life Update Mobile — runtime Gemini app

Framing · Agent-native product design

Vocabulary to use

  • Call the LLM surface a thinking surface, not "AI screen".
  • Call user entries messages that feed memory.
  • Call structured outputs artifacts (plans, recaps, summaries).

Patterns to emphasize

  • P02 Visible Thinking — expose reasoning for sensitive suggestions.
  • P07 Sources-first — when referencing the user's history, cite it.
  • P10 Memory Split — daily entries are Knowledge; user preferences are Instructions.
Headline pattern: “A journaling app that thinks with you” — Gemini is not a chatbot bolted on; it's the reasoning surface the app is built around.

Top 5 patterns dr-cv must demonstrate fluency in

  1. P01 · Artifact Panel — split-pane chat + persistent output surface (Claude, v0, Canvas).
  2. P02 · Visible Thinking — streamed, collapsible reasoning with the exact label "Thinking".
  3. P04 · Plan before Build — externalised, editable plan as the steering surface for agent runs.
  4. P05 · Diff-Review as the Unit of Trust — every AI mutation is a reviewable, rollback-able change.
  5. P09 · Named Capabilities (Skills) — tools exposed as composable, discoverable skills — not hidden features.
