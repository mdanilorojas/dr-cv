02Library cage match

Six libraries, one solo dev, no co-founder. Bundle cost is gzipped over-the-wire. Feasibility-for-dr-cv assumes Danilo has shipped React but never Three.js in production.

Three.js + R3F

~280–350 KB gzipped (three 150 + R3F 40 + drei loaders + postprocessing 80)
Learning
5 — weeks to competence, months to lighting mastery
Best use
One PBR hero object + HDRI, docked behind DOM. The raycast.com cube playbook.
Worst use
Full-site camera rigs solo. Becomes the project, not a piece of it.
ex · lusion.co · framer.com
dr-cv feasibility 2/5. Viable only if scoped to one scene, one material, one HDRI. Scope creep kills it.

OGL

~6 KB gzipped — order of magnitude smaller than three
Learning
4 — easy JS, hard GLSL (no PBR safety net)
Best use
One fullscreen fragment shader as hero: cymatics ring, iridescent plane, noise-displaced field.
Worst use
Loading glTF models. You'll rewrite three poorly.
ex · robin-noguier.com
dr-cv feasibility 3/5. Awwwards-adjacent polish at 1/40th the bundle, but demands one real GLSL skill.

Rive

~120 KB gzipped WASM + 20–200 KB per .riv
Learning
3 — editor is Figma-adjacent; state machines take a week to click
Best use
Stateful vector animations that react to hover/click/scroll. Huly's entire motion layer.
Worst use
Photoreal materials, iridescence, volumetric beams. Zero shader surface.
ex · huly.io · duolingo.com
dr-cv feasibility 4/5. The Huly-grade polish lever. Authoring time > coding time, which favors a designer-led solo dev.

GSAP (+ ScrollTrigger, SplitText, Flip)

~50 KB gzipped core; +10–15 KB per plugin
Learning
2 — API is the most polished in the space
Best use
Scroll-pinned sequences, text reveals, timeline choreography that plain CSS can't sequence.
Worst use
Tiny state transitions on a component (Framer Motion wins there).
ex · olivierlarose.com · huncwot.com
dr-cv feasibility 5/5. As of 2024 GSAP + all plugins are free for commercial. No reason not to adopt.

Framer Motion

~35 KB gzipped (tree-shakable)
Learning
1 — declarative React, identical mental model to JSX
Best use
Component-scoped animation: layout shifts, presence, drag, gesture, shared-element transitions.
Worst use
Long scroll-pinned sequences choreographed across 10+ elements. GSAP beats it there.
ex · emilkowal.ski · cursor.com
dr-cv feasibility 5/5. The Next.js-native default. Inline interactive demos (Emil's pattern) live here.

Canvas 2D (vanilla)

0 KB — browser native
Learning
2 — the API is small; the hard part is perf discipline
Best use
Particle fields, beam motes, type-writer terminals, dithered halftones. Huly's ascending particles.
Worst use
3D, lighting, materials. Wrong tool.
ex · huly.io (beam particles) · jhey.dev
dr-cv feasibility 5/5. Already referenced in huly-shader-research.md effect #1 (~1.3 KB gzipped). Free signature motion.

03What's over-used in 2026

Eight techniques that have crossed the line from "fresh" to "everyone's landing looks like this." Using them isn't wrong — but they won't earn a single compliment. They're the price of admission, not the hook.

01 · overused

Radial gradient blob behind the hero headline

The "purple-to-cyan mesh blob under the H1" is the 2024–25 equivalent of 2018's gradient button. Every AI landing has one. Cursor, v0, Vercel, Linear, Inkeep, LangChain — same move, same 700-blur, same 0.4 opacity.

ev: 7 / 9 AI sites in R2 use this. It's the default Vercel template.

02 · overused

Warm radial glow on the primary CTA pill

Cursor-following radial halo on a rounded rectangle. Huly has it, Linear has it, basement has it, a hundred Tailwind UI kits ship it. Safe but invisible in 2026. If you do it, it should behave differently on idle (breathing) than on hover (following).

ev: huly.io baseline · cursor.com · linear.app

03 · overused

Magnetic buttons

Button lunges toward cursor inside a 60px radius. Was novel in 2021 via Olivier Larose's GSAP tutorial; by 2026 it's the default "interactive" flex on awwwards portfolios. Skip unless the whole site commits to physics.

ev: noomoagency.com · basement.studio · olivierlarose.com

04 · overused

Chromatic aberration on type hover

Three-channel RGB split on letterforms during hover/scroll. Stunning once; exhausting across 20 headlines. Reads "I watched a GLSL tutorial" more than it reads "signature."

ev: activetheory.net · robin-noguier.com · kprverse.com

05 · overused

Glass morphism with backdrop-filter

Frosted translucent cards over a colored background. Apple made it prestige in 2020; by 2026 it's the fallback when no other idea shows up. Still fine for a single element (nav bar). Avoid on 5+ components.

ev: raycast.com · igloo.inc · most Tailwind starters

06 · overused

Horizontal scroll pinned galleries

Vertical scroll becomes horizontal inside a pinned section. GSAP ScrollTrigger tutorial copy-paste. Breaks accessibility, confuses mobile, and every awwwards winner 2022–2024 did it.

ev: robin-noguier.com · noomoagency.com · lusion.co

07 · overused

Full-bleed looping Lottie behind text

A 200 KB Lottie JSON of abstract shapes looping behind hero copy. Enterprise SaaS (Inkeep, Gong, Clay) all ship this. Rive does the same job with interactivity; Lottie just loops.

ev: inkeep.com · most Y Combinator S22–S25 landings

08 · overused

Terminal / IDE mock as hero screenshot

A photoshopped "IDE window" with faked code, faked status bar, faked git indicator. Cursor earned it (their product IS an IDE). Everyone else is cosplaying. If you're not shipping an editor, don't show one.

ev: cursor.com (authentic) · dozens of AI-coding clones (derivative)

04What's still fresh / underused

Eight moves that would make dr-cv distinguishable in the AI-landing crowd right now. Several are tier-2 studio techniques that the solo-AI-tool crowd hasn't copied yet.

01 · fresh

Conic-gradient aura around a single object

Animated iridescent halo encoded as conic-gradient + @property, masked to a ring. Pure CSS, zero JS. Huly uses it behind the notification bell — almost nobody else has caught on. It's also in huly-shader-research.md effect #2 (0 KB JS).

ev: huly.io · docs/references/huly-shader-research.md

02 · fresh

Ascending-mote light beam (canvas 2D)

~45 particles rising inside a CSS-drawn light cone. Zero WebGL, ~1.3 KB gzipped. Reads as Huly's signature hero without the Rive+WASM bill. Works against the "profile + agentic AI" metaphor — agents as motes in the beam.

ev: huly.io · huly-shader-research.md effect #1

03 · fresh / underused

Rive state machines driving real UI

Instead of decorative animation, hook Rive state machines into actual interactions — tab switches, status indicators, agent-activity feeds. Huly does it; no agentic-landing peer does. Fresh because Rive is still misread as a Lottie replacement.

ev: huly.io · shader-research dossier §3

04 · underused

Scroll-driven SVG draw as "agent walking the page"

stroke-dashoffset tied to IntersectionObserver. Draws a single line that threads through sections as the user scrolls — the line IS the agent's path. LangChain hints at it; nobody in our R2 set commits. Cheapest wow-per-byte on this list.

ev: langchain.com (partial) · emilkowal.ski essay embeds

05 · fresh

One PBR hero object on a plinth

Raycast's cube is the canonical reference. Single glTF, HDRI environment, cursor rotates it, no scroll-bound camera rig. Cheap at Tier-2, high at perceived quality. A physical rendering of the CV / skills-sheet as a stacked object would fit dr-cv's "artifact, not résumé" narrative.

ev: raycast.com · immersive-g.com · shader-research §1

06 · underused

Inline interactive micro-demo inside the case study

Emil Kowalski's signature: the essay about animation contains a working toast widget. For dr-cv: a case study about agent workflows contains a working mini-agent. Proof of craft lives where the claim lives. Radically underused in AI landings.

ev: emilkowal.ski · rauno.me/craft

07 · fresh

DOM-to-WebGL image upgrade (@14islands/r3f-scroll-rig)

Semantic <img> tags get lifted into WebGL planes on scroll so they can accept grain / RGB split / liquid-reveal shaders without sacrificing HTML / a11y / SEO. Still rare outside 14islands' own work. Lets dr-cv have shader effects with screen-readers intact.

ev: 14islands.com · their own open-source scroll-rig

08 · underused

Serif-display H1 in a monospace-heavy layout

Every AI landing uses Inter or Geist. A single distinctive serif (PP Editorial New, Fraunces, Migra, GT Sectra) on name + H1 instantly escapes the "ChatGPT-generated landing" visual signature. Biggest perceptual upgrade per byte (one @font-face, ~40 KB).

ev: unseen.studio · huncwot.com · henry.codes · round-04 steal #08

05Three shader setups for dr-cv, ranked by risk

Concrete build plans. Each assumes one solo dev (Danilo + agents), no prior Three.js production shipping. Bundle costs are gzipped on the wire. Time estimates include learning, scoping, and iteration — not just code.

Low risk

CSS-first + canvas 2D motif

What huly-shader-research.md already covers. No WebGL. No npm install.

Scope
Light-beam hero (canvas 2D motes + CSS cone) · conic-gradient aura around one glyph · dotted-lattice card backdrops · cursor-glow CTA buttons · scroll-driven SVG draw for the agent-path thread.
Bundle
~1.6 KB gzipped JS + 0 KB CSS deltas. Dwarfed by one product screenshot.
Time
3–5 days of design + implementation for a solo dev. Every effect is already written and tested.
Risk
Near-zero. prefers-reduced-motion gates already built. Safari / Firefox clean.
Ceiling
Huly-adjacent polish. Cannot deliver PBR metallic or true iridescence. Reads as craft, not spectacle.
Stack
CSS (conic + radial + mix-blend-mode)
canvas 2D (~45 particles)
IntersectionObserver + stroke-dashoffset
GSAP ScrollTrigger (optional, +50 KB)
Framer Motion for component-scoped transitions
No WebGL. No Rive. No Three.js.
Medium risk

Three.js + R3F single hero object

One PBR scene, one iridescent material, one HDRI. Everything else stays DOM.

Scope
Raycast-style hero: a physical rendering of the dr-cv skills-sheet / CV PDF as a stacked artifact on a plinth · cursor rotates it (no scroll-bound camera) · MeshPhysicalMaterial with iridescence: 1 + city HDRI from drei · fallback static PNG under prefers-reduced-motion + WebGL unavailable.
Bundle
~280 KB gzipped (three 150 + R3F 40 + drei env + suspense boundary). Code-splittable; not loaded on mobile.
Time
3–5 weeks cold start. Week 1: tutorials + the Bruno Simon three.js journey materials. Week 2: HDRI + material. Week 3: glTF modeling / export. Weeks 4–5: polish, mobile fallback, perf.
Risk
Medium. Main failure mode is scope creep (adding scroll-bound camera turns it into Tier 3). A strict "one scene, no postprocessing" rule keeps it sane.
Ceiling
Raycast-grade. The artifact metaphor is defensible for a CV site ("my work is a product, rotate it") in a way Raycast's decorative cube is not.
Stack
three (150 KB) + @react-three/fiber (40 KB)
@react-three/drei: Environment, Float, OrbitControls
one glTF model (~100 KB DRACO-compressed)
No postprocessing (bloom, DOF, god-rays are Tier 3 territory)
Framer Motion for the rest of the page
prefers-reduced-motion + @react-three/fiber error boundary
High risk

Multi-scene R3F + postprocessing + scroll camera

Active Theory / Lusion territory. The site becomes the case study.

Scope
Persistent R3F canvas fixed behind DOM · scene swaps per section (hero / skills-sheet / case-studies / contact) · scroll-driven camera rig via @14islands/r3f-scroll-rig · postprocessing (bloom + DOF + grain + chromatic), god-rays on beam · custom ShaderMaterial for noise-displaced plane · DOM-to-WebGL image upgrade on case thumbnails · preloader with progress · WebGPU fallback to WebGL2.
Bundle
400–550 KB gzipped once postprocessing + scroll-rig + custom shaders are wired. Mobile must load a stripped build.
Time
8–12+ weeks cold start for a solo dev. Realistically: 4–6 months to reach Awwwards-submission polish. Custom GLSL, performance budgeting across devices, and art-direction iteration all compound.
Risk
High. This IS the project, not a feature of it. Phase-2 CV generators and Phase-3 landing copy get starved. Abandonment cost is near-total — sunk shaders don't port down cleanly.
Ceiling
Lusion / Active Theory. Unambiguously tier-1. But the gap between Tier 2 and Tier 3 is 3x time for <30% perceived uplift on a CV site, where copy and case-study proof dominate conversion.
Stack
three + @react-three/fiber + @react-three/drei (loaders, env, materials)
@react-three/postprocessing (bloom, DOF, god-rays, chromatic)
@14islands/r3f-scroll-rig (DOM-to-WebGL)
Lenis (smooth scroll) + GSAP ScrollTrigger
Custom ShaderMaterial + shaderMaterial helper
HDRI environments + DRACO-compressed glTF
Plus a strict device-tier detection + mobile static fallback
verdict for dr-cv v3

Ship LOW. Earmark MEDIUM for v3.1.

The LOW-risk tier (CSS + canvas 2D + scroll-SVG) already clears every identified "ante" technique and claims four of the eight still-fresh moves: conic-gradient aura, ascending light-beam, scroll-driven SVG draw, and the serif-H1 swap. That's enough distinction against the AI-landing crowd — Cursor, Linear, v0, Vercel, Inkeep all ship far less motion than Low-tier delivers.

The MEDIUM tier is a real option for v3.1 once positioning and copy are validated with traffic. The Raycast-style PBR artifact pays off precisely because dr-cv has a tangible object to render (the skills-sheet PDF + CV variants). That metaphor is not fakeable by competitors who don't ship artifacts.

The HIGH tier is a trap for this phase. Lusion-grade output requires 4–6 months solo. In that window, Phase 2 CV generators (already shipping) and Phase 3 landing copy are the higher-leverage work. Do not ship WebGL before shipping a working landing.

Library picks for Low tier: CSS-only for conic-aura + light-beam CSS · vanilla canvas 2D (~1.3 KB) for particles · GSAP ScrollTrigger + SplitText for section choreography · Framer Motion for component transitions · one serif @font-face (Fraunces or PP Editorial New). Total JS over-the-wire: ~90 KB gzipped on top of Next.js. Everything else is HTML, CSS, and SVG.
